### Labeling Multiple Navigation Regions

When a web page has multiple navigation menus, it is critical to use ARIA labels to identify each one so users can distinguish between regions of the same type. Unique regions, such as `<main>`, do NOT need additional labels.

```html
<nav aria-label="Main">
    <ul>
        <li><a href="/dashboard">Dashboard</a></li>
        <li><a href="/profile">Profile</a></li>
    </ul>
</nav>

<nav aria-label="Related Links">
    <ul>
        <li><a href="/help">Help Center</a></li>
        <li><a href="/logout">Log Out</a></li>
    </ul>
</nav>
```

### Labeling Regions with Existing Headings

If there is visible text that labels an element, you can use `aria-labelledby` to reference the ID of that heading. This gives the region an accessible name based on existing content, which is often preferred over a hidden `aria-label`.

```html
<aside aria-labelledby="regionheading">
    <h2 id="regionheading">On this Page</h2>
    <ul>
        <li><a href="#intro">Introduction</a></li>
        <li><a href="#details">Details</a></li>
    </ul>
</aside>
```

**WCAG Considerations:**
* [WCAG 2.4.6: Headings and Labels (Level AA)](https://www.w3.org/WAI/WCAG22/Understanding/headings-and-labels.html): Providing descriptive labels (like "Main" and "Related Links" for the navs) helps users understand the purpose of the components. 
* [ARIA Landmarks Example](https://www.w3.org/WAI/ARIA/apg/patterns/landmarks/examples/navigation.html): The WAI-ARIA Authoring Practices Guide strongly recommends unique labels for multiple structural landmarks of the same type.
