---
title: Inaccessible Page Regions
description: Examples of poor page region markup and accessibility failures.
---

### Unlabelled Identical Regions

A common failure is having multiple regions of the same type without distinguishing them. If a screen reader encounters two `<nav>` elements without labels, the user will not know which one is the primary menu and which is a secondary or footer menu.

```html
<nav>
    <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
    </ul>
</nav>

<main>
    <h1>Welcome</h1>
    <p>Page content here...</p>
</main>

<nav>
    <ul>
        <li><a href="/privacy">Privacy Policy</a></li>
        <li><a href="/terms">Terms</a></li>
    </ul>
</nav>
```

*Failure Note:* This fails to provide adequate labeling for distinct landmarks. Always label regions to distinguish between regions of the same type (e.g., using `aria-label="Main"` and `aria-label="Footer"`).

### Skipping Heading Ranks

Headings communicate the organization of the content on the page and should be nested logically by their rank (1 through 6). Skipping heading ranks can be confusing and should be avoided where possible.

```html
<main>
    <h1>SpaceTeddy Inc. Products</h1>
    
    <article>
        <h2>Cotton Fur Bears</h2>
        
        <h4>Washing Instructions</h4>
        <p>Hand wash only.</p>
    </article>
</main>
```

*Note on Exceptions:* It is acceptable to skip ranks when closing subsections. For example, an `<h2>` beginning a new section can follow an `<h4>` as it closes the previous section.

**WCAG Considerations:**
* [WCAG 1.3.1: Info and Relationships (Level A)](https://www.w3.org/WAI/WCAG22/Understanding/info-and-relationships.html): Skipping heading levels breaks the semantic outline of the page, violating the logical relationship of the content.
* [WCAG 2.4.1: Bypass Blocks (Level A) Failure](https://www.w3.org/WAI/WCAG22/Understanding/bypass-blocks.html): If structural markup like landmarks or headings is confusing, absent, or poorly structured, users cannot safely navigate or bypass repetitive blocks of content.
